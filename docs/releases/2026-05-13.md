# Release 2026-05-13

## Packages

| Package | Version |
| ------- | ------- |
| `@trackdigger/web` | `0.2.0` |
| `@trackdigger/python-service` | `0.1.0` |

## Highlights

Pagination lands across search, favorites, and discography; a new `/favorites` page lets users browse saved tracks. The Bandcamp similarity adapter is removed (see ADR-0023), and two scoring fixes eliminate false positives from Cosine when the queried track isn't in its catalogue and from Discogs tracklists when a release's headline artist differs from the per-track performer.

## Changes

### New Features

- **`/favorites` page** — paginated list of saved tracks accessible from the nav (`web/features/favorite/`, `web/app/favorites/page.tsx`)
- **Server-side pagination for search and favorites** — skip/take with shared `Pagination` component; 18 tracks per page; previous page stays visible while next loads (`web/features/search/`, `web/app/api/search/[id]/route.ts`)
- **Per-request cost instrumentation** — `RequestMetric` table populated by Hono middleware + Prisma extension + FastAPI middleware (`web/lib/metrics/hono.ts`, `python-service/app/core/metrics.py`)
- **Player sub-components and keyboard shortcuts** — arrow-key prev/next track, media session seek, extracted `PlayerCover`, `PlayerPlaybackControls`, `PlayerProgressBar`, `PlayerTrackInfo`, `PlayerVolume` (`web/features/player/components/`)

### Bug Fixes

- **Cosine bare-artist fallback removed** — Cosine no longer recommends off an unrelated seed when the queried track isn't in its catalogue; reversed-order retry is kept; `SEARCH_CACHE_VERSION` bumped to `v7` (`web/features/search/searchCache.ts`, `python-service/app/api/routes/similar.py`, ADR-0024)
- **Discography track artist attribution** — tracks on a release where the searched artist is a remixer/contributor no longer have an empty artist field; the release's headline artist is used as the per-track fallback (`python-service/app/adapters/discogs.py`, `web/features/discography/server/tracklist.ts`)
- **Session validation** — session user is now verified against the DB on each request; stale JWTs trigger an automatic sign-out (`web/features/auth/server/authApi.ts`)
- **YTM exact-match** — per-token typos in `/ytm/search-exact` title matching are now tolerated (`python-service/app/api/routes/ytm_playlist.py`)

### Breaking Changes

- **Bandcamp `/similar` adapter removed** — the Python adapter, the `bandcamp` `SourceList` slot, and the Phase 2 artist fallback are gone; `SEARCH_CACHE_VERSION` bumped to `v6` in that step (then to `v7` by the Cosine fix above). The web-side Bandcamp scraper is kept as a YTM-fallback in `embed-resolver.ts`. See ADR-0023.
