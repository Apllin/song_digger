## Highlights

First stable release of Track Digger — a similarity-search tool for electronic music. Ships a full authentication system with brute-force protection, seven external source adapters fused via RRF, a Bandcamp/YouTube playback layer, and a response cache that keeps repeated searches fast.

## Changes

### New Features

**Authentication & Security**
- Auth.js v5 JWT sessions with email/password — registration, email verification (6-digit codes), login, password reset, and sign-out (`web/lib/auth.ts`, `web/app/actions/`)
- Cloudflare Turnstile CAPTCHA on registration; shown on login after 3 failed attempts
- Per-IP brute-force protection: rate limit (10 attempts / 15 min), per-email exponential backoff, email alert at 5+ failures
- Anonymous request gate: 10 free searches across `/api/search` and discography endpoints; 11th returns 429 and prompts registration
- Strict Content Security Policy, security headers, honeypot fields on auth forms, and Zod input validation on all API entry points

**Similarity Search**
- Seven adapters contributing to `/similar`: Bandcamp, Cosine.club, Last.fm, trackid.net, Yandex Music, YouTube Music, and a Last.fm artist-fallback path (`python-service/app/adapters/`)
- RRF fusion with artist diversification (max 2 consecutive) in `web/lib/aggregator.ts`
- Per-user dislike filter applied post-fusion; anonymous users see no filtering
- `ExternalApiCache` — 14-day Postgres cache keyed by search version + normalized query, covering both the Python `/similar` response and per-adapter calls

**Playback & Embeds**
- Bandcamp tracks play via a hidden `<audio>` element with a unified playback controller
- YouTube embeds loaded via the IFrame API

**Discography**
- Label and artist discography search backed by the Discogs adapter (`/api/discography/`, `python-service/app/adapters/discogs.py`)
- Typed Hono RPC client generated by kubb for the label feature folder

**Infrastructure**
- pnpm + Turborepo monorepo with `web` (Next.js 16, Prisma 7, Tailwind v4, Jotai) and `python-service` (FastAPI)
- Railway deployment via Dockerfiles; single shared `.env` at repo root
- Three-tier test suite: unit, smoke (live network), and speed (latency thresholds)
- Changesets-based release workflow with `rc-release.yml` and `release.yml` GitHub Actions

### Dependencies
