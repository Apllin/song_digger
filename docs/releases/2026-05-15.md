## Highlights

SoundCloud joins the similarity sources with full in-player playback; the player gains a paginated playlist extender that continues the queue across page boundaries and auto-skips unplayable tracks. A trainer feedback system lets users label results as similar or not, and a logistic-regression model learns per-source RRF weights from that signal.

## Changes

### New Features

- **SoundCloud source** — SoundCloud is now a similarity adapter in the Python service with an embedded SoundCloud player in the web UI; promo noise (PREMIERE/FREE DL prefixes, catalog and label suffixes) is stripped from titles (`python-service/app/adapters/soundcloud.py`, `web/features/player/hooks/useSoundCloudPlayer.ts`)
- **Trainer feedback system and ML-based RRF weight learning** — thumbs-up/down buttons on result cards write labeled samples to the DB; a `/train` endpoint fits a logistic-regression model on those samples and produces per-source RRF weights that replace the static defaults (`web/features/feedback/components/TrainerFeedbackButtons.tsx`, `python-service/app/api/routes/train.py`, `web/lib/modelWeights.ts`)
- **Paginated playlist extender** — the search queue now continues across page boundaries; when the last loaded track plays, the next page is fetched and appended automatically; the player registers an extender driven by the queue tail (`web/features/player/hooks/useNextTrackPreload.ts`, `web/features/search/hooks/useSearchFlow.ts`)
- **Album chaining in discography and labels** — after the last track of a release the player auto-loads the next release (fetching the next page when needed) and the accordion for that release auto-expands (`web/features/discography/hooks/useAlbumPlaylistExtender.ts`, `web/features/discography/components/DiscographyPage.tsx`)
- **Favorites from discography and labels** — heart button added to tracks on the Discography and Labels tabs; the source URL for a favorited discography track points to the Discogs release page (`web/features/favorite/server/favoriteAddBySource.ts`, `web/components/discography/TrackRow.tsx`)

### Bug Fixes

- **SoundCloud seed leaked into results** — the `/recommended` page embeds the seed track in its player widget so it appeared in the parsed results; it is now excluded (`python-service/app/adapters/soundcloud.py`)
- **Cosine/Yandex/YTM seed strictness** — seed search now requires an exact title match for "Artist - Title" queries and falls back to artist-only matching for bare-artist queries; the source is dropped when no candidate qualifies, eliminating recommendations off an unrelated seed (`python-service/app/adapters/_seed_match.py`, `python-service/app/adapters/cosine_club.py`)
- **Embed resolver for null source** — tracks with a null or incomplete source in discography and labels no longer stall on "No playback available"; they fall through to the YTM resolver (`web/features/player/hooks/usePlayer.ts`)
- **Auto-skip unplayable tracks** — tracks whose embed resolution returns null are auto-skipped via `onEnded` so playback continues (`web/features/player/hooks/useAudioPlayer.ts`)
- **YouTube pause on card close** — clicking the currently-playing card to close the player now pauses YouTube playback instead of leaving it running silently (`web/features/player/hooks/useYTPlayer.ts`)
- **Search SSR hydration and infinite loop** — fixed a hydration mismatch in SearchBar and an infinite `/api/search` loop triggered when opening a Find Similar URL directly (`web/features/search/components/SearchSection.tsx`, `web/features/search/server/searchApi.ts`)
